# Guidelines for a Good Business Opportunity Specification

# Objective

This specification is primarily for discovering features and artifacts that maximize business profit. Risk reduction, operational automation, and legal compliance are also profit-related because they reduce potential losses and penalties.

Given this objective, this document should focus on discovery and future prioritization based on expected business value. The specification should identify features and artifacts that achieve one or both of the objectives below.

- Increase the revenue of the business by capturing more business.
- Reduce selling costs and operating expenses by reducing or eliminating manual effort.

After this specification, the following steps should occur before implementation: prioritization, effort estimation, budgeting, detailed requirements (for example, UI and non-functional requirements), and a program specification.

# Workflow

Use the workflow below to produce the business opportunity specification, while keeping the objective above in focus.

1. Identify the major workflows of the target application and the steps that can be automated or streamlined with software.
2. For each workflow step, map relevant personas and identify artifacts that streamline operations.
Examples: consolidated reports, concise decision summaries, AI-suggested actions.
3. Note that some artifacts may be reused across multiple workflow steps or different workflows.
4. Create a dedicated section for each artifact and include:
- Purpose and benefit
- Typical user action after using it
Example: call a client to set an appointment, or identify buyers with expected funds to be released in the next 3 months.
- Data included in the artifact
- Key filters or criteria used to generate the artifact
5. State key assumptions in the specification.
6. List outstanding issues, such as performance concerns or data availability constraints.

# Checklist

- Are any features or artifacts that could increase profit being overlooked?
- Workflow steps are clearly identified.
- Workflow artifacts are clearly identified.
- Each artifact identified in step 2 shall have a corresponding sections to elaborate its function and benefit
- Specification output format is exactly the same as specified by the user.
