# Guidelines for a Good Business Specification

Use the following workflow to produce a business specification.

1. Identify the major workflows of the required application, and areas that could be automated, streamlined with a software.
2. For each workflow step relates it to personas and identify the artifacts that streamline operations.
Examples: consolidated reports, concise decision summaries, AI-suggested actions.
3. Note that some artifacts may be reused across multiple workflow steps or different workflows.
4. Create a dedicated section for each artifact and include:
- Purpose and benefit
- Typical user action after using it
Example: call a client to set an appointment or identify buyers with expected funds released in the next 3 months.
- Data included in the artifact
- Key filters or criteria used to generate the artifact
5. State key assumptions in the specification.
6. List outstanding issues, such as performance concerns or data availability constraints.

# Checklist

- Workflow steps are clearly identified.
- Workflow artifacts are clearly identified.
- Artifact details are covered in separate sections.
- Specification output format completely same as what the user specified.
